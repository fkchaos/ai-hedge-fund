# S&P500 七大佬辩论出手时机判断器 — 操作手册

## 快速开始

### 1. 命令行查询（最简单）

```bash
cd /root/ai-hedge-fund

# 仅估值查询（不辩论）
poetry run python -m src.debate.sp500_timing --brief

# 估值查询 + 自动辩论
poetry run python -m src.debate.sp500_timing

# 完整输出（含大佬观点）
poetry run python -m src.debate.sp500_timing --json
```

### 2. Python代码调用

```python
import sys
sys.path.insert(0, '/root/ai-hedge-fund')

from src.debate.sp500_timing import check_sp500_timing

# 基础用法
result = check_sp500_timing()
print(result.summary)

# 指定辩论轮数
result = check_sp500_timing(debate_rounds=3)

# 强制辩论（即使估值中性）
result = check_sp500_timing(force_debate=True)

# 自定义主题
result = check_sp500_timing(topic="当前是否应该加仓摩根标普500?")

# 自定义超时（默认30秒/次调用）
result = check_sp500_timing(call_timeout=60.0)
```

## 估值判断逻辑

### 单项指标区间
| 指标 | 🟢 便宜 | 🟡 合理 | 🟠 偏贵 | 🔴 极贵 |
|------|---------|---------|---------|---------|---------|
| S&P500 PE | ≤18 | 18-22 | 22-27 | ≥27 |
| VIX | ≥30（恐慌买入） | 25-30 | 20-25 | ≤20（贪婪） |
| 52周位置 | ≤30% | 30-50% | 50-80% | ≥80% |
| 10Y国债 | ≤2.5% | 2.5-3.5% | 3.5-4.5% | ≥4.5% |

### 综合信号
- **强烈买入** 🟢：3个以上指标显示"便宜"
- **可以分批买** 🟢：2个指标显示"便宜"
- **偏贵** 🟠：2个以上指标显示"偏贵"
- **不买** 🔴：3个以上指标显示"极贵"
- **中性** 🟡：其余情况

## 辩论触发逻辑

系统会根据估值信号自动决定是否辩论：

| 估值信号 | 自动辩论 | 辩论轮数 | 说明 |
|---------|----------|----------|------|
| 强烈买入 🟢 | ✅ 是 | 2轮 | 快速确认买入机会 |
| 可以分批买 🟢 | ✅ 是 | 2轮 | 标准辩论 |
| 偏贵 🟠 | ✅ 是 | 1轮 | 轻量辩论确认风险 |
| 不买 🔴 | ✅ 是 | 2轮 | 确认高风险 |
| 中性 🟡 | ❌ 否 | - | 无需辩论 |

使用 `force_debate=True` 可强制辩论。

## 七大佬角色

### 优化说明（2026-07-19更新）
本次优化深度还原了每位大佬的投资哲学和思维模型：
- 添加了详细的决策框架（6步分析流程）
- 添加了Few-Shot示例（教LLM如何应用框架）
- 添加了个性化特征（语言风格、常用短语）

### 1. Warren Buffett — 价值投资之王
**核心投资哲学：**
1. 以合理价格买入优秀公司，而不是以便宜价格买入平庸公司
2. 别人贪婪时我恐惧，别人恐惧时我贪婪
3. 长期持有，理想是永远
4. 不懂不做，只投能力圈内的公司
5. 护城河是最重要的——找有持久竞争优势的企业

**思维模型（6步决策）：**
1. 能力圈检验 → 2. 护城河分析 → 3. 管理层评估 → 4. 财务健康 → 5. 估值判断 → 6. 长期持有检验

**信号判断规则：**
- Bullish: 优秀公司 + 合理价格 + 安全边际>25% + 愿意持有10年
- Bearish: 平庸公司 OR 高估 OR 超出能力圈
- Neutral: 好公司但价格不合理，或信息不足

### 2. Benjamin Graham — 价值投资之父
**核心投资哲学：**
1. 市场先生是你的仆人，不是你的主人——利用他的情绪
2. 安全边际是投资的基石——永远以大幅低于内在价值的价格买入
3. 内在价值不依赖于市场价格，而是基于资产和盈利能力
4. 投资与投机要严格区分
5. 分散化保护你免受无知的伤害

**思维模型（格雷厄姆数字法）：**
1. 财务健康筛查 → 2. 估值计算（Graham Number） → 3. 安全边际评估 → 4. 持有检验

**信号判断规则：**
- Bullish: Graham Number打折30%+，财务稳健，持续盈利
- Bearish: 估值无安全边际，或财务不健康
- Neutral: 好公司但价格不够便宜

### 3. Charlie Munger — 质量投资大师，逆向思维
**核心投资哲学：**
1. 反过来想，总是反过来想
2. 多元思维模型——用多学科知识分析问题
3. 太难了就说不
4. 避免愚蠢比追求聪明更重要
5. 宁可错过十个好机会，也不接受一个坏机会

**思维模型（多元模型）：**
数学 + 心理学（25种偏误）+ 经济学 + 生物学

**决策框架：**
1. 识别愚蠢 → 2. 逆向思考 → 3. 机会成本 → 4. 太难堆 → 5. 安全边际

### 4. Peter Lynch — GARP投资之王
**核心投资哲学：**
1. 知道你拥有什么，知道你为什么拥有它
2. 买入你了解的公司
3. 用PEG估值：PE/增长率 < 1 是便宜
4. 寻找"十倍股"（ten-bagger）
5. 股价最终反映盈利增长

**思维模型（5步分析）：**
1. 公司分类 → 2. PEG检验 → 3. 故事检验 → 4. 资产负债表 → 5. 增长驱动

**信号判断规则：**
- Bullish: 合理分类 + PEG<1 + 增长故事成立 + 资产负债表健康
- Bearish: 增长放缓 OR PEG>2 OR 资产负债表恶化
- Neutral: 增长不确定或估值合理

### 5. Michael Burry — 逆向投资大师，泡沫探测器
**核心投资哲学：**
1. 市场先生是躁郁症患者——利用他的情绪
2. 找市场共识的错误——大众通常是错的
3. 风险第一，收益第二
4. 不对称性：小亏大赚
5. 数据驱动，不用直觉

**思维模型（6步逆向分析）：**
1. 识别市场共识 → 2. 找出共识的错误 → 3. 深度价值分析 → 4. 催化剂识别 → 5. 不对称性评估 → 6. 尾部风险分析

**信号判断规则：**
- Bullish: 市场共识错误 + 深度价值 + 催化剂 + 不对称性
- Bearish: 拥挤交易 OR 估值泡沫 OR 无催化剂
- Neutral: 无法量化优势

### 6. Stanley Druckenmiller — 宏观对冲大师
**核心投资哲学：**
1. 保住本金不是第一要务——找到大机会时要全力以赴
2. 流动性和宏观环境决定一切——自上而下
3. 不对称性：亏小赚大，找到3:1以上的机会
4. 趋势一旦形成，不要逆势——动量是最好的朋友
5. 知道何时防守比进攻更重要

**思维模型（宏观+动量）：**
1. 宏观环境判断 → 2. 市场情绪/动量 → 3. 不对称性评估 → 4. 仓位管理

**信号判断规则：**
- Bullish: 宏观友好 + 动量向上 + 不对称性3:1+
- Bearish: 宏观恶化 + 动量向下 + 下行风险大
- Neutral: 宏观中性或不确定

### 7. Nassim Taleb — 反脆弱大师，尾部风险专家
**核心投资哲学：**
1. 黑天鹅事件比你想象的更频繁——永远为极端事件做好准备
2. 反脆弱：从混乱和波动中获益，而不只是抵抗
3. 不对称性：确保你的上行空间远大于下行风险
4. 避免"脆弱"的系统——杠杆、集中、预测
5. 尾部对冲：买保险，即使大部分时间亏小钱

**思维模型（反脆弱+尾部风险）：**
1. 识别脆弱性 → 2. 尾部风险评估 → 3. 反脆弱机会 → 4. 凸性检验

**信号判断规则：**
- Bullish: 市场恐慌过度 + 反脆弱机会 + 凸性头寸 + 尾部风险被低估
- Bearish: 市场脆弱 + 拥挤交易 + 凹性头寸 + 尾部风险被低估
- Neutral: 不确定性太高，无法判断凸性方向

## Phase 1/2/3 辩论机制

系统采用三阶段聚焦辩论机制，在7位大佬之间进行高效决策：

### Phase 1：快速投票（Vote）
- **参与者：** 全部7位大佬
- **目的：** 每人快速给出信号 + 目标仓位
- **特点：** 不辩论，不看彼此观点，纯粹独立判断
- **输出：** 7个投票，按 bullish/bearish/neutral 分类

### Phase 2：聚焦挑战（Focused Challenge）
- **参与者：** 只选分歧最大的代表（最多4人参与）
- **选择策略：**
  - 全员一致 → 跳过Phase 2
  - 一个少数派 → 少数派 + 多数派最强辩手
  - 两方对峙（4:3） → 各方最高置信度代表
  - 三方分裂 → 各组代表
- **特点：** 聚焦核心争议，减少LLM调用次数
- **轮数：** 根据估值信号自动决定（1-2轮）

### Phase 3：最终投票（Final Vote）
- **参与者：** 全部7位大佬（含之前未参与挑战的）
- **目的：** 看到辩论结果后重新表态
- **特点：** 可以改票，也可以坚持原有观点
- **输出：** 7个最终投票 + 各大佬目标仓位

### 综合阶段（Synthesis）
- **参与者：** 辩论主持人
- **目的：** 综合所有观点得出共识
- **输出：** 共识方向、置信度、一致同意点、风险提示、行动建议

### 典型调用次数
| 场景 | Phase 1 | Phase 2 | Phase 3 | 综合 | 总计 |
|------|---------|---------|---------|------|------|
| 全员一致 | 7 | 0 | 7 | 1 | 15 |
| 有分歧（1轮挑战） | 7 | 4 | 7 | 1 | 19 |
| 有分歧（2轮挑战） | 7 | 8 | 7 | 1 | 23 |

## 量化预计算模块（quant_signals.py）

**核心理念：不让LLM猜技术指标，而是用Python精确计算后作为结构化数据喂给它。**

### 技术指标（用yfinance拉取SPY 1年历史数据计算）

| 类别 | 指标 | 说明 |
|------|------|------|
| 均线系统 | MA5/20/50/200 | 价格相对位置、均线排列（多头/空头/缠绕） |
| 动量指标 | RSI(14) | 超买（≥70）/ 超卖（≤30）/ 中性 |
| 动量指标 | MACD(12,26,9) | 金叉 / 死叉 / 零轴上方 / 零轴下方 |
| 波动率 | 布林带(20,2) | 上轨附近 / 下轨附近 / 中轨附近 |
| 波动率 | ATR(14) | 真实波动幅度，占价格百分比 |
| 成交量 | OBV + 量比 | 放量 / 缩量 / 正常 |
| 趋势强度 | ADX(14) | 强趋势（≥40）/ 中等趋势 / 弱趋势/震荡 |

### 指数估值
| 指标 | 说明 |
|------|------|
| PE(TTM) | 滚动市盈率 |
| 盈利收益率(EY) | = 1/PE × 100 |
| 10年期国债收益率 | 来自 ^TNX |
| **股权风险溢价(ERP)** | = EY - 10Y国债 |
| ERP解读 | >4% 有吸引力 / 2-4% 中性 / <2% 偏贵 |
| PE vs 历史均值 | 偏离历史均值PE≈16的百分比 |

### 综合技术评分
根据均线排列、RSI、MACD、布林带位置、量价关系等多维度打分，得出"偏多"/"偏空"/"中性"的综合判断。

## 历史记忆系统（history.py）

辩论系统具备记忆能力，能加载过往辩论记录作为上下文：

### 功能
1. **辩论存档：** 每次辩论自动保存完整记录到 `data/debate_history/` 目录
2. **历史加载：** 读取最近N次辩论（默认3次），供大佬参考
3. **记忆注入：** Phase 1投票前把历史上下文注入prompt
4. **表现追踪：** 辩论后记录SPY涨跌，评估各大佬预测准确率

### 记忆内容
- 历史辩论的市场快照（SPY价格、PE、VIX）
- 历史投票分布和最终结论
- 辩论后SPY实际表现（📈/📉）
- 各大佬的历史准确率汇总

### 防幻觉机制
- 每次辩论开始时注入**当前日期**到所有prompt中
- 确保LLM只能基于截止到当前日期的已知信息进行分析
- 防止LLM使用未来数据做出判断

## 超时保护

每次LLM调用都有独立超时保护：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `call_timeout` | 30秒 | 单次LLM调用超时 |
| 重试次数 | 1次 | 超时后自动重试 |
| 降级策略 | 自动跳过 | 连续2次失败后禁用该大佬 |

- 超时或连接错误会记录到 `AgentStatus`
- 被禁用的大佬会被跳过，不影响其他大佬
- 最终诊断会报告活跃大佬数量和跳过的大佬列表

```python
    elif result.final_signal == "🔴":
        print("❌ 不建议买入")
        print(f"   原因: {result.final_advice}")
        # 继续观望...
        
    else:
        print("⏳ 观望中")
        print(f"   信号: {result.final_advice}")
    
    # 保存结果
    with open("investment_decision.txt", "w") as f:
        f.write(result.summary)

if __name__ == "__main__":
    make_investment_decision()
```

## 常见问题

### Q1: 辩论太慢怎么办？
辩论采用三阶段机制，通常需要60-120秒（15-23次LLM调用）。如果只需要快速估值：
```bash
# 使用 --brief 只看估值结论
poetry run python -m src.debate.sp500_timing --brief
```

### Q2: 如何只看某个大佬的观点？
目前辩论会调用所有7位大佬。如需单独调用，可直接使用 `run_debate.py`：
```bash
poetry run python run_debate.py --topic "S&P500估值" --agents buffett,munger --rounds 1
```

### Q3: 某个大佬超时/失败怎么办？
系统会自动重试一次，连续2次失败后会禁用该大佬。诊断信息会显示活跃大佬数量和被跳过的大佬列表。辩论不受单个大佬故障影响。

### Q4: 数据获取失败怎么办？
如果 yfinance 获取失败，可能是网络问题。检查：
```python
import yfinance as yf
spy = yf.Ticker("SPY")
print(spy.info)
```

### Q5: 历史记录保存在哪里？
辩论历史保存在 `data/debate_history/` 目录，每次辩论自动生成一个JSON文件。系统会自动加载最近3次辩论记录作为大佬的记忆上下文。

### Q6: 目标仓位是什么意思？
每位大佬在Phase 1和Phase 3投票时会输出一个`target_position`（0-100%），表示他们认为应该在SPY中持有的仓位比例。系统取7位大佬的平均值作为最终目标仓位。0%=全现金，100%=满仓SPY。

### Q7: 如何保存结果？
```bash
# 命令行保存到文件
poetry run python -m src.debate.sp500_timing > report.txt

# JSON格式保存
poetry run python -m src.debate.sp500_timing --json > report.json
```

### Q8: 如何在cron任务中使用？
```bash
# cron job prompt 示例
cd /root/ai-hedge-fund && poetry run python -m src.debate.sp500_timing --brief
```
